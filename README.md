A full-stack data analytics mini project for Computer Engineering students.

## 🚀 Quick Start

### Step 1 — Run Python Analysis

```bash
cd python_analysis
pip install -r requirements.txt
python backend.py
```

This generates `dashboard/src/data/analytics.json`.

> **With real Kaggle data:** Download the [Olist Brazilian E-Commerce Dataset](https://www.kaggle.com/datasets/olistbr/brazilian-ecommerce) and replace the synthetic data section in `analyze.py` with:
> ```python
> orders = pd.read_csv("olist_orders_dataset.csv")
> items  = pd.read_csv("olist_order_items_dataset.csv")
> # merge and clean...
> ```

### Step 2 — Launch the Dashboard

```bash
cd frontend
npm install
npm run dev
```

Open [http://localhost:5173](http://localhost:5173) in your browser.

### Step 3 — Build for Production

```bash
npm run build
npm run preview
```

## 📊 Dashboard Sections

| Section | Charts |
|---|---|
| **Overview** | KPI Cards, Monthly Revenue, Category Bars, Payment Pie, Reviews, Hourly Orders |
| **Sales Trends** | Monthly Area Chart, Weekday Pattern, Hourly Heatmap, Discount Impact |
| **Products** | Category Performance, Top SKUs, Discount Analysis |
| **Geography** | State-wise Revenue & Orders with Progress Bars |
| **Customers** | Review Distribution, Payment Preferences, Weekly Behaviour |

## 📦 Tech Stack

| Layer | Technology |
|---|---|
| Data Analysis | Python, Pandas, NumPy |
| Frontend Framework | React 18 + Vite |
| Charts | Recharts |
| Icons | Lucide React |
| Styling | Pure CSS with CSS Variables |
| Dataset | Olist Brazilian E-Commerce (Kaggle) |

## 🎓 Learning Outcomes

- **Data wrangling** with Pandas (groupby, aggregations, time-series)
- **EDA** — revenue trends, category performance, customer behaviour
- **React component architecture** with props and state
- **Data visualization** with Recharts (Area, Bar, Pie, Composed charts)
- **JSON as a data bridge** between Python backend and JS frontend
- **Dashboard UX** — KPI cards, navigation, responsive grid layouts
